## 先别急着说 TCP 阻断——新手最容易犯的四个操作错误

遇到 SSH 连不上，很多人第一反应是"IP 被封了"，但实际上不少情况根本不是封 IP，是自己填错了信息。先把这四条核对一遍，排除了再往下查。

**1. SSH 端口不是 22**

这是新手最高频的坑。大多数 VPS 默认 SSH 端口是 22，但搬瓦工不是。搬瓦工给每台 VPS 分配的是一个随机 5 位数端口，你在 KiwiVM 控制面板里能找到，购买成功的邮件里也会写明。如果你把端口填成了 22，不管怎么连都进不去，跟 IP 有没有被封没有任何关系。

**2. 用户名必须是全小写 root**

`root` 就是 `root`，不是 `Root`，也不是 `ROOT`。SSH 对用户名大小写敏感，填错了就是认证失败，直接报错拒绝连接。

**3. 密码是系统重装后发到邮件的那个**

如果你重装过系统，密码已经变了。新密码在 KiwiVM 控制面板里可以直接修改，重装完系统之后记得去更新一下 SSH 客户端里保存的密码。

**4. VPS 没开机**

进 KiwiVM 面板确认一下 VPS 的状态是不是 Running。如果是 Stopped 状态，当然连不上，启动一下就行了。

---

上面四条都确认没问题了，那大概率就是 TCP 阻断的问题了。

## 什么是 TCP 阻断：为什么 ping 通了 SSH 还不行

这里稍微解释一下底层逻辑，知道了原因之后后面的操作就好理解了。

ping 命令发的是 ICMP 报文，ICMP 是网络层协议，不走端口。而 SSH 走的是 TCP，是传输层协议，有端口号的概念（搬瓦工默认的那个 5 位随机端口）。

当国内网络对某个 IP 实施 TCP 阻断时，它只丢弃 TCP 数据包，ICMP 包照常放行。所以你 ping 那个 IP，回包正常，一切看起来都好——但一旦发起 SSH 连接，TCP 握手包直接被墙截掉，连接超时，什么响应都没有。

换端口有没有用？大部分情况没用。TCP 阻断是针对 IP 做的，不是针对某个端口，换了端口发出去的还是 TCP 包，一样会被拦。

## 正式确认：用 KiwiVM 官方工具检测 TCP 是否被阻断

搬瓦工 KiwiVM 面板自带一个 IP 封锁检测工具，这是最直接的方法。

1. 登录搬瓦工账号，进入 My Services，找到那台 VPS，点进 KiwiVM Control Panel
2. 进去之后在面板 URL 里找到你的 VPS-ID（就是 URL 里那串数字）
3. 把 URL 最后的 `main.php` 替换成 `main-exec.php?mode=blacklistcheck`，回车打开
4. 点击 **Test Main IP** 开始检测

如果结果显示 **IP BLOCKED**，那就确认了，IP 已经被列入封锁名单，TCP 阻断是主因，SSH 自然连不上。

如果显示 **IP NOT BLOCKED**，那 IP 本身没问题，再去排查本地防火墙或网络环境的问题。

需要注意的是，换 IP 之后如果网络套餐线路本身质量不好，过一段时间可能还会再被封。这种情况下，换 IP 只是治标。

### 方式二：换机房（CN2 GIA-E 套餐可用）

如果你买的是 CN2 GIA-E 套餐，这个套餐支持在 12 个机房之间切换，切换机房之后 IP 会变，相当于免费换了一个 IP。在 KiwiVM 面板里找 Migration 菜单，选个目标机房，一两分钟搞定，数据不丢。

但有一个限制：**IP 已经被封的状态下，换机房这个操作可能会被系统限制**，不一定每次都能成功。能换成功是赚到，换不成功就得走付费换 IP 的路子。

### 方式三：等待自然解封（不推荐但有人用）

被封之后什么都不做，等着解封，一般需要 2 到 3 个月，有时候更长。这个方法适合不急用的情况。关机或者重装系统对解封速度没有太大帮助，主要看运气。
